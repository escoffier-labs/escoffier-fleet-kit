# Fleet Illustration & Image Style Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a small, tested banner-prompt system in `escoffier-fleet-kit` that makes every README banner share the og-card brand and is structurally immune to the hexagonal reference-chain drift, then re-render the off-style banners with it.

**Architecture:** Mirror the existing `og/` pipeline (data JSON + builder `.mjs`). A `banner/style.json` holds the shared constants (palette, mandatory exclusion clause, frozen-anchor list). A `banner/briefs.json` holds one creative brief per repo. `banner/build-prompt.mjs` assembles the exact GPT Image 2 prompt for a slug and writes a provenance sidecar. The image itself is generated by GPT Image 2 (via Codex or OpenClaw) using that prompt plus exactly one frozen clean anchor, then gated at thumbnail size before it replaces the repo's banner. The deterministic parts (prompt assembly, anchor immutability, brief validation) are unit-tested; the image quality is gated by visual review.

**Tech Stack:** Node ESM (`type: module`), Node built-in test runner (`node --test`, no new deps), `sha256sum`, GPT Image 2 via Codex CLI / OpenClaw.

## Global Constraints

- Brand palette, exact values, never re-invent: ink `#0d1014`, amber `#e0a45c`, cream paper `#f5f2ea`, slate-navy ink `#1d2733`. (From `DESIGN.md`.)
- Two accents maximum per image: the amber plus at most one cool accent (teal/cyan or violet).
- Kicker grammar: `[ NN · PROJECT · CATEGORY ]`, monospaced amber caps.
- README banner aspect ~21:9 (~2048x877); subject 40-55% of frame, off-center; at least 30% bare paper/wash; readable at 280px.
- The unifying constant is the kitchen FRAME (palette + type + kicker + watercolor treatment), NOT the mascot. Mascots stay per-project and apt; never force a crustacean. Confirmed-correct mascots: code-search = llama (ollama), vervet = vervet monkey, agentpantry = lobster chef.
- Frozen anchor set is immutable: `jellyfin`, `usage-tracker`, `tokenjuice`. A generated/shipped banner MUST NEVER enter the anchor set.
- og-cards stay code-rendered (HTML to PNG via `og/render.mjs`); never AI-generated.
- No new runtime dependencies. Tests use `node --test`.
- Commit messages: conventional, human, no AI/tool attribution, no `Co-Authored-By`. No em dashes in committed prose.
- New banner generation path: GPT Image 2 (no style-weight slider, no negative-prompt field, no seed). Control is: style in text, at most one anchor as image input, exclusion clause appended to the prompt, provenance logged.

```js
import assert from 'node:assert/strict';
import { readdirSync, readFileSync } from 'node:fs';
import { createHash } from 'node:crypto';
import { fileURLToPath } from 'node:url';
import { dirname, join } from 'node:path';

const dir = join(dirname(fileURLToPath(import.meta.url)), 'anchors-v1');

test('exactly the three frozen anchors are present', () => {
  const names = readdirSync(dir)
    .filter((f) => /\.(jpg|png)$/i.test(f))
    .map((f) => f.replace(/\.(jpg|png)$/i, ''))
    .sort();
  assert.deepEqual(names, ['jellyfin', 'tokenjuice', 'usage-tracker']);
});

test('checksums match SHA256SUMS (immutability guard)', () => {
  const lines = readFileSync(join(dir, 'SHA256SUMS'), 'utf-8').trim().split('\n');
  assert.ok(lines.length === 3, 'expected three checksum lines');
  for (const line of lines) {
    const [hash, name] = line.trim().split(/\s+/);
    const actual = createHash('sha256').update(readFileSync(join(dir, name))).digest('hex');
    assert.equal(actual, hash, `checksum drift on ${name}`);

});
```

- [ ] **Step 3: Write `banner/build-prompt.mjs`**

```js
// Assemble the GPT Image 2 prompt for a repo banner from shared style + per-project brief.
// Usage: node banner/build-prompt.mjs <slug>     (print prompt, write provenance sidecar)
//        node banner/build-prompt.mjs --all       (every slug)
import { readFileSync, writeFileSync } from 'node:fs';
import { fileURLToPath } from 'node:url';
import { dirname, join } from 'node:path';

const here = dirname(fileURLToPath(import.meta.url));
const reposRoot = join(here, '..', '..');
const defaultStyle = JSON.parse(readFileSync(join(here, 'style.json'), 'utf-8'));
const defaultBriefs = JSON.parse(readFileSync(join(here, 'briefs.json'), 'utf-8'));

export function buildPrompt(slug, briefs = defaultBriefs, style = defaultStyle) {
  const b = briefs[slug];
  if (!b) throw new Error(`No brief for slug: ${slug}`);
  if (!style.anchors.includes(b.anchor)) throw new Error(`Brief ${slug} uses unknown anchor: ${b.anchor}`);
  const subject = b.character ? `A ${b.character} ${b.action}.` : `A loose watercolor flatlay: ${b.action}.`;
  const accent = b.accentCool
    ? `Warm amber accent plus a single cool ${b.accentCool} accent; no third hue.`
    : 'Warm amber accent only; keep it nearly monochrome cream and ink.';
  const composition = `Composition: subject ${style.frame.subjectCoverage}; at least ${style.frame.negativeSpaceMin} bare paper; aspect ${style.frame.aspect}.`;
  const kicker = `Small monospaced amber caps top-left reading "[ ${b.category} ]", with clear space around it.`;
  return [subject, style.styleLanguage, accent, composition, kicker, style.exclusionClause].join(' ');
}

export function provenance(slug, briefs = defaultBriefs, style = defaultStyle) {
  const b = briefs[slug];
  return [
    'model: gpt-image-2',
    'generated_via: codex|openclaw',
    `slug: ${slug}`,
    `anchor: ${style.anchorDir}/${b.anchor}.jpg`,
    `prompt: ${buildPrompt(slug, briefs, style)}`,
  ].join('\n') + '\n';
}

if (import.meta.url === `file://${process.argv[1]}`) {
  const arg = process.argv[2];
  if (!arg) { console.error('usage: node banner/build-prompt.mjs <slug>|--all'); process.exit(1); }
  const slugs = arg === '--all' ? Object.keys(defaultBriefs) : [arg];
  for (const slug of slugs) {
    const sidecar = join(reposRoot, defaultBriefs[slug].target) + '.prompt.txt';
    console.log(`\n=== ${slug} ===\n${buildPrompt(slug)}`);
    writeFileSync(sidecar, provenance(slug));
    console.log(`(provenance -> ${sidecar})`);
  }
}
```

- [ ] **Step 4: Create `ILLUSTRATION.md`**

```markdown
# Escoffier Labs Illustration & Image Style

Canonical reference for the fleet's images. Companion to `DESIGN.md` (the web
design system). Source of truth for constants is `banner/style.json`; this doc
explains the intent.

## Principle: the frame is the constant, the animal is the variable

Brand cohesion comes from the kitchen frame (palette, type, kicker grammar,
loose-watercolor treatment), not from any one mascot. Mascots are per-project
and apt. This keeps the fleet harness-agnostic: the lobster is OpenClaw's
mascot and must not become the universal fleet mascot.

## Shared brand DNA

- Palette (from `DESIGN.md`): ink `#0d1014`, amber `#e0a45c`, cream `#f5f2ea`,
  slate-navy ink `#1d2733`. Two accents maximum per image.
- Type: Inter (display), IBM Plex Mono (kickers/labels), Caveat (paper only).
- Kicker: `[ NN · PROJECT · CATEGORY ]`, amber mono caps, top-left.

## Two artifacts, one DNA

- og-card: code-rendered (`og/render.mjs`, HTML to PNG). Text-led. Never
  AI-generated. This is why it never drifts.
- README banner: GPT Image 2 watercolor. Character-led, one mascot in the
  frame, open wash background. Built via `banner/`.

## Why banners used to drift (the hexagonal pattern)

Reusing generated banners as image references made the model latch onto
watercolor granulation plus literal background grids (machine walls, node
meshes) as one tiled-cell feature and regularize it toward honeycomb, worse
each generation. The fix is the frozen anchor set plus the exclusion clause.

## Generating a banner
```
